# OpenAgentTrace v0.1

**Status:** Draft | **Version:** 0.1.0 | **Date:** 2026-04-29

---

## Overview

OpenAgentTrace is a semantic convention extension for OTel that describes the
lifecycle of AI agents and their interactions. It is additive to the existing
[OTel GenAI semantic conventions](https://opentelemetry.io/docs/specs/semconv/gen-ai/)
and the [OpenInference specification](https://github.com/Arize-AI/openinference).

Tracelane implements OpenAgentTrace natively. Any OTel collector that can receive
OTLP can consume OpenAgentTrace spans without modification.

## Span kinds

### 1. `agent.run`

The top-level span for a single agent invocation. Contains the entire agent lifecycle.

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| `agent.name` | string | вң… | Agent identifier (e.g. `code-reviewer`) |
| `agent.run_id` | string | вң… | Unique run identifier (UUID v7) |
| `agent.framework` | string | вң… | Framework: `anthropic`, `openai-assistants`, `langchain`, `langgraph`, `autogen`, `crewai`, `custom` |
| `agent.model` | string | вң… | Primary model (e.g. `claude-sonnet-4-6`) |
| `agent.goal` | string | вқҢ | Human-readable goal or task description |
| `agent.step_count` | int | вқҢ | Total steps taken in the run |
| `agent.budget_tokens` | int | вқҢ | Max token budget if set |
| `agent.budget_spent_tokens` | int | вқҢ | Actual tokens spent |
| `agent.outcome` | string | вқҢ | `success`, `failure`, `timeout`, `budget_exceeded`, `blocked` |

### 2. `agent.step`

A single step within an agent run вҖ” one iteration of the plan-execute-observe loop.

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| `agent.step.index` | int | вң… | 0-indexed step number within the run |
| `agent.step.state` | string | вң… | State: `planning`, `executing`, `observing`, `reflecting` |
| `agent.step.tools_called` | string[] | вқҢ | Tool names called in this step |
| `agent.step.decision` | string | вқҢ | `continue`, `complete`, `retry`, `abort` |

### 3. `llm.call`

A single LLM API call. Uses OTel GenAI conventions plus OpenInference extensions.

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| `gen_ai.system` | string | вң… | Provider: `anthropic`, `openai`, `google` |
| `gen_ai.request.model` | string | вң… | Requested model ID |
| `gen_ai.response.model` | string | вқҢ | Actual model used (may differ from requested) |
| `llm.model_name` | string | вң… | Model name (OpenInference) |
| `llm.token_count.prompt` | int | вң… | Input tokens |
| `llm.token_count.completion` | int | вң… | Output tokens |
| `llm.token_count.total` | int | вқҢ | Sum |
| `llm.input_messages` | json | вқҢ | Serialized input messages (redacted if PII) |
| `llm.output_messages` | json | вқҢ | Serialized output messages |
| `llm.thinking` | string | вқҢ | Extended thinking / chain-of-thought content |
| `llm.cache_read_tokens` | int | вқҢ | Anthropic cache read tokens |
| `llm.cache_creation_tokens` | int | вқҢ | Anthropic cache creation tokens |
| `llm.latency_to_first_token_ms` | int | вқҢ | Streaming TTFT |

### 4. `tool.call`

A tool invocation within an agent step.

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| `tool.name` | string | вң… | Tool/function name |
| `tool.call_id` | string | вң… | Provider-assigned call ID |
| `tool.input` | json | вң… | Serialized tool arguments (redacted if PII) |
| `tool.output` | json | вқҢ | Tool result (redacted if sensitive) |
| `tool.error` | string | вқҢ | Error message if tool failed |
| `tool.duration_ms` | int | вқҢ | Execution time |
| `mcp.server_name` | string | вқҢ | MCP server name if this is an MCP tool |
| `mcp.server_version` | string | вқҢ | MCP server version at call time |

### 5. `mcp.tool_list`

A `tools/list` call to an MCP server. Used for rug-pull detection (PP-PR1).

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| `mcp.server_name` | string | вң… | MCP server name |
| `mcp.server_version` | string | вң… | Server version at list time |
| `mcp.tools_count` | int | вң… | Number of tools returned |
| `mcp.tools_hash` | string | вң… | SHA256 of the sorted tool names list |
| `tracelane.mcp_rugpull_detected` | bool | вқҢ | True if tools changed since last list |

## Tracelane-specific attributes (`tracelane.*`)

These are additive to the OTel and OpenInference conventions.

| Attribute | Type | Description |
|-----------|------|-------------|
| `tracelane.tenant_id` | string | Tenant identifier (from JWT) |
| `tracelane.intervention` | string | `none`, `warn`, `block` |
| `tracelane.aft_ids` | string[] | AFT failure mode IDs that fired |
| `tracelane.pii_redacted` | bool | Whether PII redaction was applied |
| `tracelane.prompt_injection_detected` | bool | Whether prompt injection was detected |
| `tracelane.cost_usd` | float | Estimated provider cost in USD |

## Conformance levels

| Level | Requirements |
|-------|-------------|
| **Basic** | `agent.run` span with `agent.name`, `agent.framework`, `agent.model` |
| **Standard** | Basic + `agent.step` spans + `llm.call` spans with token counts |
| **Full** | Standard + `tool.call` spans + `mcp.tool_list` spans + `tracelane.*` attributes |

Tracelane emits at Full conformance for native SDK integrations.
